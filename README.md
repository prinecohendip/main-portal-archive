# main-portal-archive

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的统一存储与展示空间。

## 项目简介

本仓库旨在收集、归档并托管一组独立的 HTML 页面。这些页面可能包含信息展示、工具界面、说明文档或其他类型的静态内容。仓库本身不提供搜索、索引或动态功能，仅作为一个稳定的页面存档点。

## 目录说明

```
main-portal-archive/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（图片、样式、脚本等）
├── index.html      # 仓库根目录入口页面（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件或子目录对应一个独立页面，命名清晰以便识别。
- `assets/`：如需共享资源，可放置于此，页面引用时使用相对路径。
- `index.html`：可选的根入口页面，用于快速浏览或导航。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖后端服务或数据库。
- 页面之间相互独立，无强制跳转或依赖关系。
- 归档内容以静态形式保存，不保证实时更新或维护。
- 页面可能包含外部资源（如 CDN 库），但建议尽量内联或本地化以保持长期可用性。

## 维护说明

- 本仓库由维护者根据需求不定期更新，无固定发布周期。
- 新增页面请放入 `pages/` 目录，并遵循已有命名规范。
- 如删除或修改页面，请同步更新 `index.html` 中的导航链接（如有）。
- 不建议对已归档页面进行频繁修改，以保持存档的原始性。

## 使用方式

可通过以下方式访问归档页面：

1. 直接打开 `pages/` 目录下的 HTML 文件。
2. 若部署为 GitHub Pages，则可通过对应 URL 访问 `pages/` 下的页面。
3. 本地克隆仓库后，使用浏览器打开任意 HTML 文件即可。
